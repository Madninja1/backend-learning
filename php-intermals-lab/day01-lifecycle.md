## Тема

Базовое понимание, как PHP обрабатывает запрос

---

## Полный путь запроса

Client (browser)
↓
Nginx
↓
PHP-FPM
↓
Zend Engine
↓
Response

---

## 1. Браузер (Client)

Браузер — это точка входа.

Что происходит:

* пользователь вводит URL
* браузер формирует HTTP-запрос
* отправляет его на сервер

Пример:
GET /index.php HTTP/1.1

---

## 2. Nginx

Что делает:

* принимает HTTP-запрос
* определяет тип запроса

Если это:

* html / css / js → отдаёт сам
* PHP → отправляет в PHP-FPM

Важно:
nginx — это веб-сервер, а не интерпретатор

---

## 3. PHP-FPM

Что делает:

* принимает запрос от nginx
* выбирает свободный PHP-процесс
* передаёт ему PHP-файл

Важно:
PHP-FPM не исполняет код
Он управляет процессами

---

## 4. Zend Engine

Это ядро PHP.

Что делает:

* читает PHP-код
* компилирует его
* выполняет код
* возвращает результат

---

## 5. Response

После выполнения:

* формируется ответ (HTML / JSON)
* отправляется обратно в nginx
* nginx отдаёт его браузеру

---

# Ключевые вопросы + ответы

## 1. Что происходит, когда пользователь открывает сайт?

Ответ:

Браузер отправляет HTTP-запрос на сервер.
Сервер (nginx) принимает запрос и решает, как его обработать.
Если это PHP — запрос передаётся в PHP-FPM.
PHP-FPM передаёт код в Zend Engine, где он выполняется.
Результат возвращается пользователю.

---

## 2. В чём разница между nginx и PHP?

Ответ:

nginx:

* веб-сервер
* работает с HTTP
* отдаёт файлы

PHP:

* язык программирования
* исполняется через Zend Engine
* генерирует результат (HTML, JSON)

---

## 3. Зачем нужен PHP-FPM?

Ответ:

PHP-FPM управляет PHP-процессами.

Он:

* держит пул процессов
* распределяет запросы
* ускоряет обработку

Без него:

* каждый запрос запускал бы PHP заново
* это было бы медленно

---

## 4. Где выполняется PHP-код?

Ответ:

PHP-код выполняется внутри Zend Engine.

Цепочка:
PHP-FPM → Zend Engine → выполнение

---

## 5. Что возвращается пользователю?

Ответ:

Результат выполнения PHP:

* HTML (страница)
* JSON (API)
* или другой ответ

Браузер получает ответ и отображает его.

# Мини-итог

PHP не работает сам по себе.
Он всегда выполняется как часть цепочки: браузер → сервер → PHP → ответ.
Главные роли:

* nginx — принимает запрос
* PHP-FPM — управляет процессами
* Zend Engine — исполняет код
