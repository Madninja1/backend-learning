# Day 7 — PHP Internals (Arrays as HashTable)

file: php-internals-lab/day07-arrays.md

## Тема

Как устроены массивы в PHP (HashTable)

---

## 1. Важно понять

В PHP массив — это НЕ обычный массив.

Это:

HashTable (хеш-таблица)

---

## 2. Что это значит

Массив хранит:

ключ → значение

Пример:

```php
$arr = [
    0 => "a",
    1 => "b",
    2 => "c"
];
```

или:

```php
$arr = [
    "name" => "Daniil",
    "age" => 25
];
```

---

## 3. Почему это важно

В других языках:

* массив = список (индексы 0,1,2...)

В PHP:

* массив = map (как словарь)

---

## 4. Как работает HashTable

Когда ты пишешь:

```php
$arr["name"] = "Daniil";
```

PHP:

1. берёт ключ "name"
2. считает hash
3. находит место в таблице
4. кладёт значение

---

## 5. Доступ к элементу

```php
$arr["name"];
```

PHP:

1. считает hash ключа
2. быстро находит значение

👉 поэтому доступ обычно O(1)

---

## 6. Поддержка ключей

PHP поддерживает:

* int
* string

Важно:

```php
$arr["1"] и $arr[1]
```

→ это один и тот же ключ

---

## 7. Внутренняя структура (упрощённо)

HashTable:

* buckets (ячейки)
* key (ключ)
* value (zval)
* hash

---

## 8. Пример

```php
$arr = [
    "a" => 10,
    "b" => 20
];
```

Внутри:

* hash("a") → bucket → 10
* hash("b") → bucket → 20

---

## 9. Почему массивы дорогие

Потому что:

* хранят hash
* хранят ключи
* хранят zval
* много памяти на каждый элемент

---

## 10. Copy-On-Write для массивов

```php
$a = [1,2,3];
$b = $a;
```

* один HashTable
* refcount = 2

```php
$b[0] = 100;
```

* создаётся новый HashTable

---

# Ключевые вопросы + ответы

## 1. Что такое массив в PHP?

Ответ:

Это HashTable (хеш-таблица), а не обычный список.

---

## 2. Почему доступ к элементу быстрый?

Ответ:

Потому что используется hash:

ключ → hash → bucket → значение

Это даёт примерно O(1)

---

## 3. Какие ключи поддерживаются?

Ответ:

* int
* string

Причём:

"1" и 1 — это один ключ

---

## 4. Почему массивы занимают много памяти?

Ответ:

Потому что каждый элемент хранит:

* ключ
* hash
* zval
* служебные данные

---

## 5. Что происходит здесь?

```php
$a = [1,2,3];
$b = $a;
```

Ответ:

* один HashTable
* refcount = 2
* копии нет

---

## 6. А здесь?

```php
$b[0] = 100;
```

Ответ:

* создаётся новый HashTable
* $a остаётся старым
* $b получает копию

---

## 7. Почему PHP использует HashTable?

Ответ:

Потому что это даёт:

* быстрый доступ
* гибкость (любые ключи)
* удобство работы

---

## 8. В чём минус такого подхода?

Ответ:

* больше памяти
* медленнее, чем чистый массив (как в C)

---

## 9. Что произойдёт здесь?

```php
$arr = [];
$arr[0] = "a";
$arr["0"] = "b";
```

Ответ:

* это один ключ
* значение будет "b"

---

## 10. Почему это важно для backend?

Ответ:

Потому что:

* массивы используются везде
* понимание структуры → понимание производительности
* можно избегать лишних затрат памяти

---

# Мини-схема

ключ → hash → bucket → zval

---

# Мини-итог

Массив в PHP — это HashTable, а не простой список.
Он хранит пары ключ-значение и обеспечивает быстрый доступ через hash.
Это делает PHP гибким, но увеличивает потребление памяти.
Copy-On-Write применяется и к массивам, что позволяет оптимизировать работу.
