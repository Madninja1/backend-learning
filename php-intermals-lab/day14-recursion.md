# Day 14 — PHP Internals (Recursion & Stack Overflow)

file: php-internals-lab/day14-recursion.md

## Тема

Как работает рекурсия и почему возникает переполнение стека

---

## 1. Что такое рекурсия

Рекурсия — это когда функция вызывает саму себя.

---

## Пример

```php id="8p3f9n"
function f($n) {
    if ($n == 0) return;
    f($n - 1);
}
```

---

## 2. Как это работает

Каждый вызов:

* создаёт новый stack frame
* добавляется в Call Stack

---

## Пример стека

```php id="3h7x2q"
f(3)
f(2)
f(1)
f(0)
```

---

## 3. Base case (базовый случай)

Это условие остановки рекурсии.

```php id="0v5w8z"
if ($n == 0) return;
```

Без него:

→ бесконечная рекурсия

---

## 4. Раскрутка стека

После достижения base case:

```id="i1k9cs"
f(0) → завершение
f(1) → завершение
f(2) → завершение
f(3) → завершение
```

---

## 5. Пример с возвратом

```php id="x2d8mf"
function sum($n) {
    if ($n == 1) return 1;
    return $n + sum($n - 1);
}
```

Вызов:

```id="c3l7zp"
sum(3)
= 3 + sum(2)
= 3 + 2 + sum(1)
= 3 + 2 + 1
= 6
```

---

## 6. Stack Overflow

Если слишком много вызовов:

```php id="v6k2dr"
function f() {
    f();
}
```

→ стек переполняется
→ ошибка

---

## 7. Почему стек ограничен

Потому что:

* стек — это ограниченная память
* каждый frame занимает место

---

## 8. Глубина рекурсии

Зависит от:

* памяти
* настроек PHP
* размера frame

---

## 9. Рекурсия vs цикл

### Рекурсия

* удобнее для сложных задач
* но дороже по памяти

### Цикл

* быстрее
* экономит память

---

## 10. Когда использовать рекурсию

* деревья
* графы
* divide & conquer

---

# Ключевые вопросы + ответы

## 1. Что такое рекурсия?

Ответ:

Это вызов функцией самой себя.

---

## 2. Что такое base case?

Ответ:

Условие, при котором рекурсия останавливается.

---

## 3. Что произойдёт без base case?

Ответ:

Бесконечная рекурсия → stack overflow

---

## 4. Что происходит в стеке при рекурсии?

Ответ:

Каждый вызов создаёт новый stack frame

---

## 5. Что такое stack overflow?

Ответ:

Это ошибка, когда стек переполняется из-за слишком большого числа вызовов

---

## 6. Что происходит здесь?

```php id="l5r2bz"
function f($n) {
    if ($n == 0) return;
    f($n - 1);
}
```

Ответ:

* создаётся цепочка вызовов
* при $n == 0 → остановка

---

## 7. Почему рекурсия дороже по памяти?

Ответ:

Потому что каждый вызов создаёт новый stack frame

---

## 8. Когда лучше использовать цикл?

Ответ:

Когда можно избежать рекурсии и снизить нагрузку на стек

---

## 9. Что делает Zend Engine при рекурсии?

Ответ:

Создаёт stack frame для каждого вызова и управляет стеком

---

## 10. Почему это важно для backend?

Ответ:

Потому что:

* влияет на производительность
* важно для алгоритмов
* помогает избегать ошибок

---

# Мини-схема

вызов → новый frame → стек растёт
base case → стек начинает очищаться

---

# Мини-итог

Рекурсия — мощный инструмент, но требует аккуратности.
Каждый вызов увеличивает стек, и без base case возникает переполнение.
Понимание стека помогает писать более эффективный и безопасный код.
