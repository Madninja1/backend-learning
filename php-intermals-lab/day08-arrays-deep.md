# Day 8 — PHP Internals (Arrays Deep: Collisions & Memory)

file: php-internals-lab/day08-arrays-deep.md

## Тема

Коллизии в HashTable и почему массивы дорогие по памяти

---

## 1. Напоминание

Массив в PHP = HashTable

ключ → hash → bucket → значение

---

## 2. Что такое hash

Hash — это число, полученное из ключа.

Пример:

```php id="z1u6y2"
hash("name") → 123456
```

---

## 3. Что такое коллизия

Коллизия — это ситуация, когда:

два разных ключа дают одинаковый hash

---

## Пример (упрощённо)

```text
hash("a") = 100
hash("b") = 100
```

→ оба попадают в один bucket

---

## 4. Как PHP решает коллизии

PHP использует:

**цепочки (linked list / open addressing)**

То есть:

* в одном bucket хранится несколько элементов
* PHP перебирает их при поиске

---

## 5. Что происходит при доступе

```php id="y4t3zi"
$arr["key"];
```

Если нет коллизии:
→ быстрый доступ (O(1))

Если есть коллизия:
→ перебор элементов (O(n))

---

## 6. Почему это важно

При большом количестве коллизий:

* доступ замедляется
* производительность падает

---

## 7. Расширение HashTable

Когда массив растёт:

* увеличивается размер таблицы
* происходит **rehashing**

---

## Что это значит

PHP:

1. создаёт новую таблицу
2. пересчитывает hash для элементов
3. переносит данные

👉 это дорогая операция

---

## 8. Память массива

Каждый элемент хранит:

* ключ
* hash
* zval
* bucket структура

---

## Пример

```php id="fw6q5c"
$arr = [1,2,3];
```

Это не просто 3 числа:

Это десятки байт на каждый элемент

---

## 9. Почему массивы “тяжёлые”

Потому что:

* универсальные (int + string ключи)
* гибкие
* хранят много метаданных

---

## 10. Copy-On-Write и память

```php id="8k0g6t"
$a = range(1, 100000);
$b = $a;
```

* один HashTable

```php id="r6k9y1"
$b[0] = 999;
```

* создаётся копия
* память удваивается

---

## 11. Реальная проблема

Если ты часто:

* копируешь массивы
* модифицируешь их

→ можешь получить сильный рост памяти

---

# Ключевые вопросы + ответы

## 1. Что такое коллизия?

Ответ:

Это ситуация, когда разные ключи дают одинаковый hash и попадают в один bucket.

---

## 2. Как PHP решает коллизии?

Ответ:

PHP хранит несколько элементов в одном bucket и перебирает их при поиске.

---

## 3. Когда доступ становится медленным?

Ответ:

Когда много коллизий → приходится перебирать элементы → O(n)

---

## 4. Что такое rehashing?

Ответ:

Это пересоздание HashTable с перераспределением элементов.

Происходит при росте массива.

---

## 5. Почему rehashing дорогой?

Ответ:

Потому что:

* создаётся новая таблица
* все элементы пересчитываются
* копируются заново

---

## 6. Почему массивы занимают много памяти?

Ответ:

Потому что каждый элемент хранит:

* ключ
* hash
* zval
* служебные данные

---

## 7. Что произойдёт здесь?

```php id="ndg8gm"
$a = range(1, 100000);
$b = $a;
```

Ответ:

* один HashTable
* refcount = 2

---

## 8. А здесь?

```php id="0y7p1o"
$b[0] = 999;
```

Ответ:

* создаётся копия массива
* память увеличивается почти в 2 раза

---

## 9. Почему это важно для backend?

Ответ:

Потому что:

* массивы используются везде
* неправильное использование → утечки памяти
* влияет на производительность

---

## 10. Когда нужно быть осторожным?

Ответ:

Когда:

* большие массивы
* частые копирования
* частые изменения

---

# Мини-схема

ключ → hash → bucket → (список элементов при коллизии)

---

# Мини-итог

Массивы в PHP — это HashTable с поддержкой коллизий.
При коллизиях производительность падает.
Каждый элемент занимает много памяти, а операции вроде rehashing и Copy-On-Write могут быть дорогими.
Понимание этого позволяет писать более эффективный backend-код.
