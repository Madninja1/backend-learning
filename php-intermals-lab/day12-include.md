# Day 12 — PHP Internals (include / require)

file: php-internals-lab/day12-include.md

## Тема

Как PHP подключает файлы и что происходит внутри include / require

---

## 1. Что такое include

```php
include "file.php";
```

Это команда:

→ взять код из файла и выполнить его

---

## 2. Что происходит внутри

Когда PHP встречает include:

1. ищет файл
2. читает его содержимое
3. пропускает через pipeline (lexer → parser → opcode)
4. выполняет код

---

## 3. include — поведение при ошибке

Если файл НЕ найден:

```php
include "no_file.php";
```

→ warning
→ выполнение продолжается

---

## 4. require

```php
require "file.php";
```

Работает так же, но:

Если файл не найден:

→ fatal error
→ скрипт останавливается

---

## 5. include_once / require_once

```php
include_once "file.php";
require_once "file.php";
```

PHP проверяет:

→ был ли файл уже подключён

Если да:

→ не подключает повторно

---

## 6. Как PHP проверяет include_once

PHP хранит список подключённых файлов:

* путь к файлу
* проверка перед подключением

---

## 7. Стоимость include_once

Минус:

* дополнительная проверка
* немного медленнее

---

## 8. Область видимости

```php
$a = 10;
include "file.php";
```

Переменные доступны внутри include

---

### Пример

file.php:

```php
echo $a;
```

→ выведет 10

---

## 9. include внутри функции

```php
function test() {
    $a = 5;
    include "file.php";
}
```

→ $a доступна внутри include

---

## 10. include возвращает значение

```php
$result = include "file.php";
```

Если в файле есть:

```php
return 42;
```

→ $result = 42

---

## 11. OPcache и include

Если включён OPcache:

* файл компилируется один раз
* opcode кэшируется
* include работает быстрее

---

## 12. Проблемы include

* дублирование кода
* неожиданные переменные
* сложность отладки

---

# Ключевые вопросы + ответы

## 1. Что делает include?

Ответ:

Он подключает файл и выполняет его код.

---

## 2. В чём разница между include и require?

Ответ:

include:

* warning при ошибке
* выполнение продолжается

require:

* fatal error
* выполнение останавливается

---

## 3. Что делают include_once и require_once?

Ответ:

Они подключают файл только один раз.

---

## 4. Почему include_once медленнее?

Ответ:

Потому что PHP проверяет, был ли файл уже подключён.

---

## 5. В какой области видимости выполняется include?

Ответ:

В текущей области видимости.

Переменные доступны внутри файла.

---

## 6. Что произойдёт здесь?

```php
$a = 10;
include "file.php";
```

Ответ:

file.php получит доступ к $a

---

## 7. Что произойдёт здесь?

```php
$result = include "file.php";
```

Ответ:

Если в file.php есть return → результат вернётся

---

## 8. Как OPcache влияет на include?

Ответ:

Он кэширует opcode, поэтому include выполняется быстрее.

---

## 9. Когда использовать require?

Ответ:

Когда файл обязателен для работы (например, конфигурация)

---

## 10. Почему include может быть опасен?

Ответ:

Потому что:

* может подтянуть неожиданный код
* создаёт скрытые зависимости
* усложняет архитектуру

---

# Мини-схема

include
↓
загрузка файла
↓
компиляция (или OPcache)
↓
выполнение

---

# Мини-итог

include и require подключают и выполняют PHP-файлы.
Разница между ними — в обработке ошибок.
include_once добавляет защиту от повторного подключения, но стоит дороже.
Понимание include важно для архитектуры и производительности приложений.
