# Что такое zval

*zval* - это внутренняя структура PHP (в Zend Engine), 
которая хранит:

* значение переменоой
* тип значения (int, string, array и т.д.)
* служебную информацию (например, для управления памятью)

Ключевая идея:

### Любая переменная в PHP - это zval

## Почему это важно

PHP — динамически типизированный язык, поэтому:

* тип может меняться
* движок обязан всегда знать тип значения

Пример:
```php
$x = 5;      // int
$x = "hello"; // string
```

**Без zval PHP не смог бы корректно 
работать с динамической типизацией**


# Что такое copy on write

Copy-on-Write — это механизм оптимизации памяти:

PHP **не копирует данные сразу при присваивании**

PHP отслеживает, используется ли значение в 
нескольких переменных, 
и копирует его только при попытке изменить.

### Пример
```php
$a = [1, 2, 3];
$b = $a;
```
### На этом этапе:

**нет реальной копии массива**

$a и $b указывают на один и тот же zval 
(или значение в памяти)

Когда происходит копия
```php
$b[0] = 100;
```
Вот здесь:

PHP создаёт **реальную копию массива**

потому что данные изменяются

## Короткая формула (запомнить)
присвоение → без копии
изменение → создаётся копия
