# Day 26 — PHP Internals (Performance Basics)

file: php-internals-lab/day26-performance.md

## Тема

Где PHP тратит ресурсы и как это связано с internals

---

## 1. Главная идея

Производительность PHP зависит от:

* памяти
* CPU
* количества операций

---

## 2. Основные “дорогие” операции

### 1. Массивы

* HashTable
* много памяти
* дорогие операции

---

### 2. Copy-On-Write

```php id="a1k9m3"
$a = range(1, 100000);
$b = $a;
$b[0] = 1;
```

→ создаётся копия
→ затраты памяти и CPU

---

### 3. Строки

```php id="v7x2c8"
$str .= "a";
```

→ новая строка каждый раз
→ дорого

---

### 4. HashTable операции

* вставка
* поиск
* rehashing

---

### 5. GC

* поиск циклов
* очистка памяти

---

## 3. Пример проблемы

```php id="t9m4k2"
$result = [];

foreach ($data as $item) {
    $result[] = $item;
}
```

→ создаются новые элементы
→ рост памяти

---

## 4. Циклы vs функции

Функции:

* создают stack frame
* чуть дороже

Циклы:

* быстрее

---

## 5. include / require

Без OPcache:

* дорого

С OPcache:

* быстрее

---

## 6. Autoload

* вызывается при загрузке классов
* может быть узким местом

---

## 7. Объекты vs массивы

Объекты:

* меньше копирования
* стабильнее

Массивы:

* гибкие
* но тяжелее

---

## 8. Где PHP быстрый

* простые операции
* работа с числами
* небольшие структуры

---

## 9. Где PHP медленный

* большие массивы
* частые копии
* строки
* GC

---

## 10. Реальный пример

```php id="p3k8x1"
$data = range(1, 1000000);
```

→ много памяти

```php id="m2v7z9"
$data = array_map(fn($x) => $x * 2, $data);
```

→ ещё больше памяти

---

## 11. Оптимизации

* избегать лишних копий
* использовать ===
* не злоупотреблять ссылками
* использовать OPcache

---

## 12. Важно понять

PHP:

* не медленный сам по себе
* медленно — неправильное использование

---

# Ключевые вопросы + ответы

## 1. Что больше всего влияет на производительность PHP?

Ответ:

* массивы
* копирование (COW)
* строки
* GC

---

## 2. Почему массивы дорогие?

Ответ:

Потому что это HashTable с большим количеством метаданных

---

## 3. Что происходит при Copy-On-Write?

Ответ:

Создаётся копия данных при изменении

---

## 4. Почему строки дорогие?

Ответ:

Потому что создаются новые строки при изменении

---

## 5. Когда работает GC?

Ответ:

При наличии циклических ссылок

---

## 6. Что быстрее: цикл или функция?

Ответ:

Цикл быстрее, потому что нет создания stack frame

---

## 7. Как OPcache влияет на производительность?

Ответ:

Убирает этап компиляции

---

## 8. Что произойдёт здесь?

```php id="n7k3x5"
$a = range(1, 100000);
$b = $a;
$b[0] = 1;
```

Ответ:

Создаётся копия массива → рост памяти

---

## 9. Почему autoload может быть проблемой?

Ответ:

Потому что вызывает загрузку файлов

---

## 10. Почему это важно для backend?

Ответ:

Потому что:

* влияет на скорость API
* влияет на память
* важно для highload

---

# Мини-схема

операция → ресурсы (CPU + RAM) → производительность

---

# Мини-итог

Производительность PHP зависит от работы с памятью и структур данных.
Массивы, строки и Copy-On-Write — основные источники затрат.
Понимание internals позволяет писать быстрый и эффективный код.
