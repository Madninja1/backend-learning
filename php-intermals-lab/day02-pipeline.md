# Day 2 — PHP Internals (Compilation Pipeline)

## Тема

Как PHP превращает код в выполнение (pipeline)

---

## Общая схема

PHP code
↓
Lexer
↓
Parser
↓
AST
↓
Opcode
↓
Zend VM (execution)

---

## 1. PHP код

Это исходный код, который пишет разработчик.

Пример:

```php
$a = 5 + 10;
```

---

## 2. Lexer (лексер)

Что делает:

* разбивает код на токены

Пример токенов:

```
$a
=
5
+
10
;
```

Важно:
Lexer не понимает смысл кода
Он просто делит его на части

---

## 3. Parser (парсер)

Что делает:

* анализирует токены
* строит структуру программы

Результат:
AST (Abstract Syntax Tree)

---

## 4. AST (абстрактное синтаксическое дерево)

Это структура, которая описывает код.

Пример:

```
   =
  / \
$a   +
     / \
    5  10
```

Важно:
AST показывает, как связаны операции

---

## 5. Opcode

AST компилируется в opcode — инструкции для выполнения

Пример:

```
LOAD_CONST 5
LOAD_CONST 10
ADD
ASSIGN $a
```

---

## 6. Zend VM

Что делает:

* исполняет opcode
* выполняет программу

---

# Ключевые вопросы + ответы

## 1. Зачем нужен Lexer?

Ответ:

Lexer разбивает код на токены.

Без него:

* невозможно анализировать код
* парсер не сможет работать

Он делает первый шаг обработки кода.

---

## 2. Что делает Parser?

Ответ:

Parser берёт токены и строит структуру программы (AST).

Он:

* проверяет синтаксис
* понимает, что означает код

---

## 3. Что такое AST?

Ответ:

AST — это дерево, которое показывает структуру кода.

Оно:

* отражает порядок операций
* показывает зависимости

Это промежуточный этап перед выполнением.

---

## 4. Что такое opcode?

Ответ:

Opcode — это набор инструкций для выполнения.

PHP-код не исполняется напрямую.
Он сначала превращается в opcode, который понимает Zend Engine.

---

## 5. Где происходит выполнение кода?

Ответ:

В Zend VM (часть Zend Engine).

Именно там:

* исполняются opcode
* выполняется программа

---

## 6. Почему PHP не исполняет код напрямую?

Ответ:

Потому что:

* исходный код неудобен для выполнения
* его нужно привести к более простому виду

Поэтому используется pipeline:

код → структура → инструкции → выполнение

---

# Мини-схема

PHP code
↓
Lexer
↓
Parser
↓
AST
↓
Opcode
↓
Zend VM

---

# Мини-итог

PHP сначала разбирает код, затем превращает его в внутреннее представление (AST), после чего компилирует в opcode и только потом выполняет.
Это делает выполнение быстрее и структурированнее.
