# Day 15 — PHP Internals (Opcode Execution & Zend VM)

file: php-internals-lab/day15-opcode.md

## Тема

Как Zend Engine исполняет PHP-код через opcode и виртуальную машину

---

## 1. Напоминание pipeline

PHP code
↓
Lexer
↓
Parser
↓
AST
↓
Opcode
↓
Zend VM (выполнение)

---

## 2. Что такое opcode

Opcode — это низкоуровневые инструкции, которые выполняет Zend VM.

PHP-код НЕ исполняется напрямую.

---

## Пример

```php id="k2x8v1"
$a = 5 + 10;
```

Превращается в:

```text id="q9m3za"
LOAD_CONST 5
LOAD_CONST 10
ADD
ASSIGN $a
```

---

## 3. Что такое Zend VM

Zend VM — это виртуальная машина PHP.

Она:

* читает opcode
* выполняет инструкции
* управляет выполнением

---

## 4. Как происходит выполнение

Zend VM:

1. берёт opcode
2. выполняет его
3. переходит к следующему

---

## 5. Execution loop

Упрощённо:

```text id="u7g2hj"
while (есть opcode) {
    выполнить opcode
}
```

---

## 6. Типы операций

Основные opcode:

* LOAD_CONST
* ASSIGN
* ADD / SUB / MUL
* FETCH (доступ к переменной)
* CALL (вызов функции)
* RETURN

---

## 7. Пример сложнее

```php id="y4n2xp"
$a = 2;
$b = 3;
$c = $a + $b;
```

Opcode:

```text id="3f8kdz"
LOAD_CONST 2 → $a
LOAD_CONST 3 → $b
ADD $a, $b → temp
ASSIGN temp → $c
```

---

## 8. Временные значения

Zend VM использует:

* временные переменные (tmp)
* стек выполнения

---

## 9. Почему это быстрее

Потому что:

* opcode проще, чем исходный код
* меньше проверок
* быстрее интерпретировать

---

## 10. OPcache

Без OPcache:

* pipeline выполняется каждый раз

С OPcache:

* opcode кэшируется
* выполнение быстрее

---

## 11. Как Zend VM работает с памятью

Во время выполнения:

* создаёт zval
* управляет refcount
* применяет COW

---

## 12. Почему это важно

Понимание opcode даёт:

* понимание производительности
* понимание “дорогих” операций
* понимание работы PHP под капотом

---

# Ключевые вопросы + ответы

## 1. Что такое opcode?

Ответ:

Это инструкции, в которые компилируется PHP-код перед выполнением.

---

## 2. Почему PHP не исполняет код напрямую?

Ответ:

Потому что opcode проще и быстрее выполнять.

---

## 3. Что делает Zend VM?

Ответ:

Она исполняет opcode и управляет выполнением программы.

---

## 4. Как происходит выполнение?

Ответ:

Zend VM проходит по списку opcode и выполняет их по очереди.

---

## 5. Что такое execution loop?

Ответ:

Это цикл, который выполняет opcode один за другим.

---

## 6. Что происходит здесь?

```php id="x9n7k1"
$a = 5 + 10;
```

Ответ:

* создаются opcode
* Zend VM выполняет их
* результат записывается в $a

---

## 7. Что дают временные переменные?

Ответ:

Они хранят промежуточные результаты вычислений.

---

## 8. Как OPcache влияет на выполнение?

Ответ:

Он кэширует opcode, чтобы не пересобирать их каждый раз.

---

## 9. Где происходит управление памятью?

Ответ:

Внутри Zend Engine во время выполнения opcode.

---

## 10. Почему это важно для backend?

Ответ:

Потому что:

* помогает понимать производительность
* помогает писать более эффективный код
* даёт глубокое понимание работы PHP

---

# Мини-схема

PHP code

выполнение

---

# Мини-итог

PHP-код сначала компилируется в opcode, а затем выполняется виртуальной машиной Zend VM.
Это делает выполнение быстрее и управляемым.
Понимание opcode помогает видеть, как реально исполняется код.
