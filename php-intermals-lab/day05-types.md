# Day 5 — PHP Internals (Data Types inside zval)

file: php-internals-lab/day05-types.md

## Тема

Как разные типы данных хранятся внутри zval

---

## 1. Типы данных в PHP

Основные типы:

* int
* float
* string
* bool
* array
* object
* null

---

## 2. Как zval хранит тип

Каждый zval содержит:

* value (значение)
* type (тип данных)

Пример:

```php
$a = 5;
```

zval:

* value = 5
* type = int

---

## 3. Примитивные типы (scalar types)

### int / float / bool

Хранятся прямо внутри zval.

Пример:

```php
$a = 42;
$b = 3.14;
$c = true;
```

Особенности:

* маленькие
* быстрые
* копируются дёшево

---

## 4. Сложные типы (complex types)

### string / array / object

Хранятся через указатель (pointer).

То есть zval хранит не само значение, а ссылку на него.

---

## 5. Строки (string)

Пример:

```php
$a = "hello";
```

zval содержит:

* указатель на строку
* длину строки
* refcount

Важно:

* строки могут шариться между переменными
* работает Copy-On-Write

---

## 6. Массивы (array)

Пример:

```php
$a = [1,2,3];
```

Внутри:

* HashTable
* ключ → значение

Важно:

* массив — это не список, а map
* хранит ключи и значения

---

## 7. Объекты (object)

Пример:

```php
$obj = new stdClass();
```

zval содержит:

* указатель на объект
* object handle

Важно:

* объекты передаются по ссылке (почти всегда)
* нет Copy-On-Write как у массивов

---

## 8. Разница: value vs pointer

### Примитивы

```php
$a = 5;
$b = $a;
```

* значение копируется быстро

---

### Сложные типы

```php
$a = [1,2,3];
$b = $a;
```

* копии нет сразу
* используется COW

---

# Ключевые вопросы + ответы

## 1. Какие типы данных есть в PHP?

Ответ:

int, float, bool, string, array, object, null

---

## 2. Как zval хранит тип?

Ответ:

zval хранит:

* значение (value)
* тип (type)

Тип определяет, как интерпретировать value

---

## 3. В чём разница между примитивами и сложными типами?

Ответ:

Примитивы:

* хранятся прямо в zval
* маленькие
* быстрые

Сложные типы:

* хранятся через указатель
* используют память отдельно

---

## 4. Как хранятся строки?

Ответ:

Через указатель:

* zval → string structure
* есть refcount
* используется Copy-On-Write

---

## 5. Почему массив — это не просто список?

Ответ:

Потому что PHP array — это HashTable:

* ключ → значение
* поддерживает int и string ключи
* не является обычным массивом как в C

---

## 6. Как ведут себя объекты?

Ответ:

Объекты:

* передаются по ссылке
* не копируются как массивы
* имеют object handle

---

## 7. Что происходит здесь?

```php
$a = "hello";
$b = $a;
```

Ответ:

* один string в памяти
* refcount = 2
* копии нет

---

## 8. А здесь?

```php
$a = [1,2,3];
$b = $a;
$b[0] = 100;
```

Ответ:

* сначала shared массив
* при изменении → копия (COW)

---

## 9. Почему это важно?

Ответ:

Понимание типов даёт:

* контроль памяти
* понимание производительности
* меньше багов

---

# Мини-схема

Примитив:
zval → value

Сложный тип:
zval → pointer → структура в памяти

---

# Мини-итог

В PHP все переменные — это zval.
Примитивные типы хранятся напрямую, а сложные — через указатели.
Массивы и строки используют Copy-On-Write, а объекты ведут себя как ссылки.
Это основа понимания памяти и производительности в PHP.
