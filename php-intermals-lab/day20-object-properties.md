# Day 20 — PHP Internals (Object Properties Storage)

file: php-internals-lab/day20-object-properties.md

## Тема

Как PHP хранит свойства объектов и получает к ним доступ

---

## 1. Свойства объекта

Пример:

```php
class User {
    public $name;
    public $age;
}

$user = new User();
$user->name = "Daniil";
```

---

## 2. Где хранятся свойства

Свойства НЕ лежат в zval напрямую.

Структура:

zval → pointer → object → properties

---

## 3. Внутри объекта

Объект содержит:

* class entry (описание класса)
* таблицу свойств (properties storage)

---

## 4. Properties storage

Это структура, где хранятся значения свойств.

По сути:

ключ → значение (zval)

---

## 5. Пример

```php
$user->name = "Daniil";
```

Внутри:

* key = "name"
* value = zval("Daniil")

---

## 6. Как PHP находит свойство

```php
$user->name;
```

PHP:

1. находит объект
2. смотрит class definition
3. находит offset свойства
4. получает значение

---

## 7. Оптимизация (offset)

Для объявленных свойств:

* PHP НЕ ищет по строке
* использует offset (индекс)

Это быстрее, чем HashTable

---

## 8. Dynamic properties

```php
$user->city = "Stockholm";
```

Если свойства нет в классе:

→ создаётся динамически

---

## Где хранится

* в HashTable внутри объекта

---

## 9. Разница

### Объявленные свойства

* быстрый доступ (offset)
* фиксированная структура

### Динамические свойства

* через HashTable
* медленнее

---

## 10. Visibility

```php
private $name;
protected $age;
```

PHP хранит:

* модификатор доступа
* проверяет его при обращении

---

## 11. Доступ к свойствам

```php
$user->name;
```

Zend Engine:

* проверяет visibility
* возвращает значение

---

## 12. Copy-On-Write и свойства

Свойства:

* не используют COW как массивы
* работают через объект

---

## 13. Память

Объект хранит:

* class entry
* properties storage
* zval для каждого свойства

---

# Ключевые вопросы + ответы

## 1. Где хранятся свойства объекта?

Ответ:

Внутри объекта, в properties storage.

---

## 2. Что содержит объект?

Ответ:

* class entry
* таблицу свойств

---

## 3. Как PHP находит свойство?

Ответ:

Через class definition и offset (для объявленных свойств)

---

## 4. Что такое offset?

Ответ:

Это индекс свойства внутри структуры объекта.

---

## 5. Чем отличаются объявленные и динамические свойства?

Ответ:

Объявленные:

* быстрые
* через offset

Динамические:

* через HashTable
* медленнее

---

## 6. Что произойдёт здесь?

```php
$user->city = "Stockholm";
```

Ответ:

Создаётся динамическое свойство в HashTable

---

## 7. Как работает visibility?

Ответ:

PHP проверяет доступ (private/protected/public) при обращении

---

## 8. Используют ли свойства Copy-On-Write?

Ответ:

Нет, они работают через объект

---

## 9. Почему доступ к свойствам быстрый?

Ответ:

Потому что используется offset вместо поиска по ключу

---

## 10. Почему это важно для backend?

Ответ:

Потому что:

* объекты используются везде
* влияет на производительность
* важно для архитектуры и моделей

---

# Мини-схема

zval → object → properties storage
→ class entry

---

# Мини-итог

Свойства объекта хранятся внутри структуры объекта, а не в zval.
Объявленные свойства доступны быстро через offset, а динамические — через HashTable.
Понимание этого помогает писать более эффективный и предсказуемый код.
