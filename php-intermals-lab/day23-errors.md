# Day 23 — PHP Internals (Errors & Error Handling)

file: php-internals-lab/day23-errors.md

## Тема

Как PHP обрабатывает ошибки: notice, warning, fatal error

---

## 1. Что такое ошибка в PHP

Ошибка — это проблема во время выполнения кода.

---

## 2. Основные типы ошибок

### Notice

* не критичная ошибка
* код продолжает выполняться

---

### Warning

* проблема серьёзнее
* код всё ещё выполняется

---

### Fatal Error

* критическая ошибка
* выполнение останавливается

---

## 3. Примеры

### Notice

```php id="n1k3v7"
echo $a;
```

→ переменная не определена

---

### Warning

```php id="p4m9x2"
include "no_file.php";
```

→ файл не найден

---

### Fatal Error

```php id="x7t2c5"
undefined_function();
```

→ выполнение остановится

---

## 4. Как PHP обрабатывает ошибки

Zend Engine:

* обнаруживает ошибку
* генерирует сообщение
* решает: продолжать или остановиться

---

## 5. error_reporting

Позволяет управлять ошибками:

```php id="c8v2p1"
error_reporting(E_ALL);
```

---

## 6. display_errors

```php id="r5n7k3"
ini_set("display_errors", 1);
```

* показывает ошибки на экране

---

## 7. Логирование ошибок

```php id="t9w3m6"
ini_set("log_errors", 1);
```

Ошибки пишутся в лог

---

## 8. set_error_handler

Можно перехватить ошибки:

```php id="y2k6x4"
set_error_handler(function($errno, $errstr) {
    echo "Error: $errstr";
});
```

---

## 9. Error vs Exception

Error:

* ошибка PHP (warning, fatal)

Exception:

* объект, который можно обработать

---

## 10. Fatal error и shutdown

Fatal error нельзя поймать обычным способом, но можно:

```php id="q7m2k9"
register_shutdown_function(function() {
    $error = error_get_last();
});
```

---

## 11. Почему это важно

Ошибки:

* влияют на стабильность
* могут ломать приложение
* важны для дебага

---

# Ключевые вопросы + ответы

## 1. Какие типы ошибок есть в PHP?

Ответ:

* notice
* warning
* fatal error

---

## 2. Чем отличается warning от fatal error?

Ответ:

warning → выполнение продолжается
fatal → выполнение останавливается

---

## 3. Что делает error_reporting?

Ответ:

Определяет, какие ошибки показывать

---

## 4. Что делает display_errors?

Ответ:

Показывает ошибки на экране

---

## 5. Что произойдёт здесь?

```php id="m2p8c1"
echo $a;
```

Ответ:

notice (переменная не определена)

---

## 6. Можно ли перехватить ошибки?

Ответ:

Да, через set_error_handler

---

## 7. Чем ошибка отличается от exception?

Ответ:

ошибка — системная
exception — управляемая

---

## 8. Можно ли поймать fatal error?

Ответ:

Нет напрямую, только через shutdown function

---

## 9. Почему важно логировать ошибки?

Ответ:

Чтобы анализировать проблемы в проде

---

## 10. Почему это важно для backend?

Ответ:

Потому что:

* ошибки влияют на стабильность
* важны для мониторинга
* критичны в production

ошибка → Zend Engine → сообщение → продолжение / остановка

PHP обрабатывает ошибки на уровне Zend Engine.
