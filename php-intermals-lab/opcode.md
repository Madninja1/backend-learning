## Что такое Zend Engine

**Zend Engine** - это ядро интерпретатора PHP.
Он отвечает за **компиляцию и выполнение PHP-кода**.

Основные функции

1. компилирует PHP-код
2. превращает его в **opcode**
3. выполняет эти инструкции
4. управляет памятью
5. управляет переменными и типами

Фактически:

**Zend Engine содержит виртуальную машину (Zend VM).**

Он работает примерно как JVM у Java.

## Как PHP выполняет код

Когда сервер получает запрос, происходит цепочка:
```
HTTP request
↓
nginx / apache
↓
php-fpm
↓
Zend Engine
↓
response
```

Но внутри Zend Engine есть ещё этапы.

## Этапы выполнения PHP

1. **Lexer (лексический анализ)**

PHP-код разбивается на **токены**.

Пример:
```php
$a = 5 + 10;
```

Токены будут примерно такие:
```
VARIABLE($a)
ASSIGN
NUMBER(5)
PLUS
NUMBER(10)
SEMICOLON
```

То есть код превращается в **структурированные элементы**.

2. **Parser (синтаксический анализ)**

Parser проверяет:

корректность синтаксиса

строит **AST — Abstract Syntax Tree**

Пример структуры:
```
ASSIGN
├── VARIABLE($a)
└── ADD
├── 5
└── 10
```

Это дерево описывает **логику программы**.

3. **Компиляция в opcode**

После AST код компилируется в **opcode**.

Opcode — это **простые инструкции для виртуальной машины**.

Пример:
```php
$a = 5 + 10;
```

Превращается примерно в:
```
LOAD_CONST 5
LOAD_CONST 10
ADD
ASSIGN $a
```

Это уже **инструкции уровня машины PHP**.

4. **Что такое Opcode**

**Opcode (operation code)** — это элементарная операция.

Каждый opcode — одна простая команда.

Примеры opcode в PHP:
```
ADD
SUB
ASSIGN
JMP
FETCH_VAR
ECHO
```

Zend Engine выполняет их **по очереди**.

Это похоже на:

* байткод Java
* инструкции процессора

Но они выполняются **виртуальной машиной PHP**.

5. **Zend VM**

После компиляции Zend Engine запускает **Zend VM (virtual machine)**.

Она:

1. берёт opcode
2. выполняет его 
3. переходит к следующему

Пример:
```
LOAD_CONST 5
LOAD_CONST 10
ADD
ASSIGN $a
```

VM выполнит:
```
stack.push(5)
stack.push(10)
stack.pop + stack.pop
result → $a
```

6. **Почему это важно**

Понимание opcode объясняет:
* почему некоторые конструкции PHP медленнее
* зачем нужен **OPcache**
* как работают оптимизации

Например:

OPcache **сохраняет opcode**, чтобы не компилировать PHP каждый запрос.
