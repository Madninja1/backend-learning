# Day 13 — PHP Internals (Functions & Call Stack)

file: php-internals-lab/day13-functions.md

## Тема

Как PHP вызывает функции и как работает стек вызовов

---

## 1. Что такое функция

Пример:

```php id="b1q2mz"
function sum($a, $b) {
    return $a + $b;
}
```

Функция — это блок кода, который можно вызвать.

---

## 2. Вызов функции

```php id="4y8x9t"
$result = sum(2, 3);
```

Что происходит:

1. создаётся новый стек-фрейм
2. передаются аргументы
3. выполняется код
4. возвращается результат

---

## 3. Что такое Call Stack

Call Stack — это стек вызовов функций.

Он хранит:

* текущие функции
* их аргументы
* локальные переменные

---

## 4. Как работает стек

Пример:

```php id="h9s2dj"
function a() {
    b();
}

function b() {
    c();
}

function c() {
    echo "done";
}

a();
```

Стек:

```
a()
b()
c()
```

После выполнения:

```
c() → удаляется
b() → удаляется
a() → удаляется
```

---

## 5. Stack frame

Каждый вызов функции создаёт frame:

Содержит:

* аргументы
* локальные переменные
* адрес возврата

---

## 6. Передача аргументов

```php id="y3s8ff"
function test($x) {
    $x = 100;
}
```

* передаётся копия (через COW)

---

## 7. Передача по ссылке

```php id="1w3nfd"
function test(&$x) {
    $x = 100;
}
```

* передаётся ссылка
* изменения влияют на оригинал

---

## 8. Возврат значения

```php id="x4w2kt"
function get() {
    return [1,2,3];
}
```

* возвращается zval
* может использовать COW

---

## 9. Рекурсия

Функция может вызывать сама себя:

```php id="l9k2as"
function f($n) {
    if ($n == 0) return;
    f($n - 1);
}
```

Каждый вызов:

* новый frame
* новый уровень стека

---

## 10. Переполнение стека

Если слишком много вызовов:

→ stack overflow

---

## 11. Zend Engine и функции

Zend Engine:

* создаёт stack frame
* управляет вызовами
* освобождает память после выполнения

---

# Ключевые вопросы + ответы

## 1. Что происходит при вызове функции?

Ответ:

Создаётся stack frame, передаются аргументы, выполняется код, возвращается результат.

---

## 2. Что такое Call Stack?

Ответ:

Это структура, которая хранит текущие вызовы функций.

---

## 3. Что хранится в stack frame?

Ответ:

* аргументы
* локальные переменные
* адрес возврата

---

## 4. Как работает стек?

Ответ:

LIFO (Last In First Out):

последняя вызванная функция выполняется первой

---

## 5. Как передаются аргументы?

Ответ:

По умолчанию — копия (через COW)

---

## 6. Что делает & в аргументах?

Ответ:

Передаёт ссылку вместо копии

---

## 7. Что происходит при рекурсии?

Ответ:

Каждый вызов создаёт новый stack frame

---

## 8. Почему возникает stack overflow?

Ответ:

Потому что стек ограничен по размеру

---

## 9. Что делает Zend Engine при вызове функции?

Ответ:

Он создаёт frame, управляет выполнением и освобождает память

---

## 10. Почему это важно для backend?

Ответ:

Потому что:

* помогает понимать выполнение кода
* важно для оптимизации
* важно для рекурсии и производительности

---

# Мини-схема

вызов функции
↓
создание stack frame
↓
выполнение
↓
return
↓
удаление frame

---

# Мини-итог

Каждый вызов функции создаёт новый stack frame.
Все вызовы хранятся в Call Stack и выполняются по принципу LIFO.
Понимание этого помогает разбираться в выполнении кода, рекурсии и производительности.
