# Day 4 — PHP Internals (zval + Copy-On-Write + Refcount)

## Тема

Как PHP хранит данные и управляет памятью

---

## 1. Что такое zval

zval — это основная структура в PHP, в которой хранится значение переменной.

Каждая переменная в PHP — это zval.

---

## Структура zval (упрощённо)

zval содержит:

* value (значение)
* type (тип: int, string, array и т.д.)
* refcount (счётчик ссылок)
* is_ref (флаг ссылки)

---

## Пример

```php
$a = 5;
```

Создаётся zval:

* value = 5
* type = int
* refcount = 1

---

## 2. Refcount (счётчик ссылок)

Показывает, сколько переменных указывают на одно значение.

---

### Пример

```php
$a = "hello";
$b = $a;
```

Что происходит:

* создаётся один zval ("hello")
* $a и $b указывают на него

refcount = 2

---

## 3. Copy-On-Write (COW)

PHP не копирует данные сразу.
Он копирует только при изменении.

---

### Пример

```php
$a = [1,2,3];
$b = $a;
```

Что происходит:

* один zval (массив)
* refcount = 2
* копии нет

---

### Изменение

```php
$b[0] = 100;
```

Теперь:

* создаётся копия массива для $b
* refcount разделяется

Итого:

* $a → старый массив
* $b → новый массив

---

## 4. Shared zval

Когда несколько переменных указывают на одно значение — это shared zval.

---

### Пример

```php
$a = 10;
$b = $a;
$c = $b;
```

* один zval
* refcount = 3

---

## 5. Почему это важно

Copy-On-Write:

* экономит память
* ускоряет работу

Без него:

* каждое присваивание копировало бы данные
* это было бы очень медленно

---

# Ключевые вопросы + ответы

## 1. Что такое zval?

Ответ:

zval — это структура, в которой PHP хранит переменные.

Она содержит:

* значение
* тип
* служебную информацию (refcount)

---

## 2. Что показывает refcount?

Ответ:

refcount показывает, сколько переменных ссылаются на одно значение.

Если:
$a = $b

→ refcount увеличивается

---

## 3. Когда происходит копирование данных?

Ответ:

Копирование происходит только при изменении (Copy-On-Write).

Пока данные не изменяются:

* используется один zval

При изменении:

* создаётся копия

---

## 4. Что такое Copy-On-Write?

Ответ:

Это механизм, при котором данные копируются только в момент изменения.

Это позволяет:

* экономить память
* избегать лишних копий

---

## 5. Что такое shared zval?

Ответ:

Это один zval, на который указывают несколько переменных.

---

## 6. Что произойдёт здесь?

```php
$a = [1,2,3];
$b = $a;
$c = $b;

$b[1] = 10;
```

Ответ:

До изменения:

* один массив
* refcount = 3

После изменения:

* $b получает копию массива
* $a и $c остаются со старым

Итого:

* $a → [1,2,3]
* $c → [1,2,3]
* $b → [1,10,3]

---

## 7. Почему Copy-On-Write важен?

Ответ:

Потому что он:

* уменьшает использование памяти
* ускоряет выполнение
* делает PHP эффективным при работе с массивами

---

# Мини-схема

$a = [1,2,3]
$b = $a

↓ (shared zval)

изменение $b

↓ (копирование)

$a → старый массив
$b → новый массив

---

# Мини-итог

В PHP переменные — это не просто значения, а структуры (zval).
Несколько переменных могут указывать на одно значение через refcount.
Копирование происходит только при изменении благодаря Copy-On-Write, что делает PHP эффективным по памяти и скорости.
