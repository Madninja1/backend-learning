# Day 22 — PHP Internals (Type Casting & Type Juggling)

file: php-internals-lab/day22-casting.md

## Тема

Как PHP приводит типы (type juggling) и как работает явное приведение

---

## 1. Что такое приведение типов

Приведение типов — это изменение типа значения.

В PHP есть:

* неявное (automatic)
* явное (explicit)

---

## 2. Неявное приведение (type juggling)

PHP автоматически меняет типы.

---

## Пример

```php id="q2m7x4"
$result = "10" + 5;
```

Что происходит:

* строка "10" → int 10
* результат = 15

---

## 3. Где происходит неявное приведение

* арифметика
* сравнение (==)
* условия (if)

---

## Пример

```php id="x9c2p6"
if ("0") {
    echo "true";
}
```

→ не выполнится (false)

---

## 4. Явное приведение

```php id="m5n8k3"
$a = (int) "10";
```

Тип явно задаётся разработчиком

---

## Основные приведения

```php id="d4p7y2"
(int)
(float)
(string)
(bool)
(array)
(object)
```

---

## 5. Примеры

```php id="t3k6v9"
(int) "123"    // 123
(int) "abc"    // 0
(bool) 0       // false
(bool) 1       // true
(string) 10    // "10"
```

---

## 6. Truthy / Falsy значения

Falsy:

```text id="y7m4d2"
false
0
0.0
""
"0"
null
[]
```

Все остальные → true

---

## 7. Проблемы type juggling

```php id="c8n3k1"
"abc" + 5  // 5
```

→ "abc" → 0

---

## 8. Сравнение и приведение

```php id="v2p6r4"
"0" == false  // true
```

→ оба приводятся к false

---

## 9. Внутри Zend Engine

Zend:

* определяет тип
* вызывает соответствующее преобразование
* работает с zval

---

## 10. Безопасная практика

* использовать ===
* избегать неявного приведения
* явно указывать типы

---

# Ключевые вопросы + ответы

## 1. Что такое type juggling?

Ответ:

Это автоматическое приведение типов в PHP.

---

## 2. Когда происходит неявное приведение?

Ответ:

* арифметика
* сравнение
* условия

---

## 3. Что такое явное приведение?

Ответ:

Когда разработчик сам задаёт тип через (int), (string) и т.д.

---

## 4. Что произойдёт здесь?

```php id="z1x8k2"
"10" + 5
```

Ответ:

строка станет числом → результат 15

---

## 5. Что произойдёт здесь?

```php id="p7m4c9"
"abc" + 5
```

Ответ:

"abc" → 0 → результат 5

---

## 6. Что такое falsy значения?

Ответ:

Значения, которые приводятся к false

---

## 7. Что произойдёт здесь?

```php id="n6t3k5"
(bool) "0"
```

Ответ:

false

---

## 8. Почему type juggling опасен?

Ответ:

Потому что может давать неожиданные результаты

---

## 9. Как писать безопасный код?

Ответ:

* использовать ===
* избегать неявных преобразований

---

## 10. Почему это важно для backend?

Ответ:

Потому что:

* влияет на логику
* может вызывать баги
* важно для безопасности

---

# Мини-схема

значение → приведение → новый тип

---

# Мини-итог

PHP активно использует автоматическое приведение типов (type juggling).
Это удобно, но может приводить к ошибкам.
Явное приведение и строгие сравнения позволяют писать более безопасный и предсказуемый код.
