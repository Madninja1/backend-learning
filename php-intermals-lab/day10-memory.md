# Day 10 — PHP Internals (Memory Model Recap)

file: php-internals-lab/day10-memory.md

## Тема

Как PHP управляет памятью: zval, refcount, Copy-On-Write — всё вместе

---

## 1. Общая картина

В PHP память строится вокруг:

* zval (контейнер)
* value (данные)
* refcount (счётчик ссылок)
* Copy-On-Write (оптимизация)

---

## 2. Как создаётся переменная

```php
$a = 10;
```

Создаётся:

* zval
* value = 10
* type = int
* refcount = 1

---

## 3. Присваивание

```php
$b = $a;
```

Теперь:

* один zval
* refcount = 2

---

## 4. Copy-On-Write

```php
$a = [1,2,3];
$b = $a;
```

* один массив
* refcount = 2

---

### Изменение

```php
$b[0] = 100;
```

Теперь:

* создаётся копия
* $a и $b разделяются

---

## 5. Ссылки (reference)

```php
$a = 10;
$b = &$a;
```

Теперь:

* один zval
* is_ref = true
* Copy-On-Write отключён

---

## 6. Строки и массивы

### Строки

* хранятся через указатель
* используют COW

### Массивы

* HashTable
* используют COW
* дорогие по памяти

---

## 7. Когда память освобождается

Когда:

refcount = 0

→ PHP удаляет значение

---

## Пример

```php
$a = [1,2,3];
$b = $a;

unset($a);
unset($b);
```

→ память освобождается

---

## 8. Проблема циклических ссылок

```php
$a = [];
$a["self"] = &$a;
```

* refcount никогда не станет 0
* возникает утечка памяти

---

## 9. Garbage Collector (GC)

PHP использует GC для:

* поиска циклических ссылок
* очистки памяти

---

## 10. Где происходят затраты

Основные расходы:

* массивы
* строки
* копирование (COW)
* rehashing

---

## 11. Типичный сценарий

```php
$a = range(1, 100000);
$b = $a;
$c = $b;

$b[0] = 999;
```

Что происходит:

* сначала один массив (refcount = 3)
* при изменении → копия
* память растёт

---

# Ключевые вопросы + ответы

## 1. Что такое zval?

Ответ:

Это структура, в которой хранится переменная:

* значение
* тип
* refcount

---

## 2. Что показывает refcount?

Ответ:

Сколько переменных указывают на одно значение.

---

## 3. Когда происходит копирование?

Ответ:

Только при изменении (Copy-On-Write).

---

## 4. Когда память освобождается?

Ответ:

Когда refcount становится 0.

---

## 5. Что делают ссылки?

Ответ:

Связывают переменные напрямую и отключают Copy-On-Write.

---

## 6. Почему массивы дорогие?

Ответ:

Потому что:

* HashTable
* много метаданных
* много памяти на элемент

---

## 7. Что такое циклическая ссылка?

Ответ:

Когда структура ссылается сама на себя и refcount не уменьшается.

---

## 8. Зачем нужен Garbage Collector?

Ответ:

Чтобы находить и удалять циклические ссылки.

---

## 9. Что произойдёт здесь?

```php
$a = [1,2,3];
$b = $a;
$c = $b;

unset($a);
unset($b);
```

Ответ:

* refcount уменьшается
* $c остаётся единственным владельцем

---

## 10. Почему это важно для backend?

Ответ:

Потому что:

* влияет на память
* влияет на производительность
* помогает избегать утечек

---

# Мини-схема

zval → value
refcount → количество ссылок
изменение → COW → копия

refcount = 0 → освобождение памяти

---

# Мини-итог

PHP управляет памятью через zval и refcount.
Copy-On-Write позволяет избегать лишних копий.
Ссылки меняют поведение и отключают оптимизации.
Garbage Collector нужен для очистки циклических ссылок.
Понимание этого даёт контроль над производительностью и памятью.
