# Day 25 — PHP Internals (Memory Lifecycle Full)

file: php-internals-lab/day25-memory-lifecycle.md

## Тема

Полный жизненный цикл данных в PHP: создание → использование → освобождение

---

## 1. Общая картина

Жизненный цикл данных:

создание → использование → копирование (COW) → освобождение → GC (если нужно)

---

## 2. Создание значения

```php id="k3m9x2"
$a = [1,2,3];
```

Создаётся:

* zval
* HashTable (массив)
* refcount = 1

---

## 3. Присваивание

```php id="p7v2c8"
$b = $a;
```

Теперь:

* один HashTable
* refcount = 2
* копии нет

---

## 4. Использование

```php id="n4x8k1"
echo $a[0];
```

PHP:

* находит zval
* получает значение
* выводит

---

## 5. Copy-On-Write

```php id="y2m6p3"
$b[0] = 100;
```

Теперь:

* создаётся новый HashTable
* $a и $b разделяются

---

## 6. Ссылки

```php id="c9k3v1"
$c = &$a;
```

Теперь:

* один zval
* COW отключён

---

## 7. Удаление переменных

```php id="x5p2m8"
unset($a);
```

* refcount уменьшается

---

## 8. Освобождение памяти

Когда:

refcount = 0

→ PHP освобождает память

---

## 9. Пример полного цикла

```php id="t6k4x2"
$a = [1,2,3];
$b = $a;
$c = $b;

unset($a);
unset($b);
unset($c);
```

→ память освобождается

---

## 10. Циклические ссылки

```php id="r8m1v7"
$a = [];
$a["self"] = &$a;
```

* refcount не падает до 0
* требуется GC

---

## 11. Garbage Collector

GC:

* находит циклы
* освобождает память

---

## 12. Где происходят затраты

Основные расходы:

* массивы
* строки
* копирование (COW)
* GC

---

## 13. Типичный сценарий нагрузки

```php id="z1k7p3"
$a = range(1, 100000);
$b = $a;

$b[0] = 999;
```

→ создаётся копия
→ память увеличивается

---

## 14. Ошибки разработчиков

* лишние копии массивов
* ненужные ссылки
* большие структуры

---

## 15. Как писать эффективно

* избегать лишних копий
* понимать COW
* не злоупотреблять ссылками

---

# Ключевые вопросы + ответы

## 1. Какие этапы жизненного цикла данных?

Ответ:

создание → использование → копирование → удаление → GC

---

## 2. Когда происходит Copy-On-Write?

Ответ:

При изменении значения

---

## 3. Когда память освобождается?

Ответ:

Когда refcount становится 0

---

## 4. Что делает unset?

Ответ:

Уменьшает refcount

---

## 5. Что происходит здесь?

```php id="m2p9x1"
$a = [1,2,3];
$b = $a;
```

Ответ:

один массив, refcount = 2

---

## 6. А здесь?

```php id="y8k3v6"
$b[0] = 100;
```

Ответ:

создаётся копия

---

## 7. Что такое циклическая ссылка?

Ответ:

Когда объект или массив ссылается сам на себя

---

## 8. Когда нужен GC?

Ответ:

При наличии циклических ссылок

---

## 9. Почему память может расти?

Ответ:

Из-за копирования и больших структур

---

## 10. Почему это важно для backend?

Ответ:

Потому что:

* влияет на память
* влияет на производительность
* важно для highload систем

---

# Мини-схема

создание → refcount ↑
копирование → COW
unset → refcount ↓
refcount = 0 → delete
циклы → GC

---

# Мини-итог

PHP управляет памятью через zval и refcount.
Copy-On-Write позволяет экономить ресурсы, но может создавать копии при изменении.
Ссылки меняют поведение системы.
Garbage Collector нужен для удаления циклических структур.
Понимание полного цикла памяти — ключ к эффективному backend-коду.
