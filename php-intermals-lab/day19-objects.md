# Day 19 — PHP Internals (Objects Internals)

file: php-internals-lab/day19-objects.md

## Тема

Как PHP хранит и обрабатывает объекты

---

## 1. Что такое объект

Пример:

```php id="k8m3v1"
class User {
    public $name;
}

$user = new User();
```

Объект — это структура с данными и поведением.

---

## 2. Как хранится объект

zval НЕ хранит объект напрямую.

Он хранит:

* указатель на объект
* object handle (идентификатор)

---

## 3. Object handle

Каждый объект имеет уникальный ID.

```php id="z2v7r8"
$a = new stdClass();
$b = $a;
```

* $a и $b указывают на один объект
* handle одинаковый

---

## 4. Объекты передаются по ссылке

```php id="h5x2k1"
$a = new stdClass();
$b = $a;

$b->x = 10;
```

Теперь:

* $a->x = 10
* $b->x = 10

---

## 5. Отличие от массивов

Массивы:

* используют Copy-On-Write

Объекты:

* НЕ копируются
* всегда по ссылке

---

## 6. Копирование объекта

```php id="f7n2q3"
$a = new stdClass();
$b = clone $a;
```

Теперь:

* два разных объекта
* разные handle

---

## 7. Внутренняя структура

Объект содержит:

* свойства (properties)
* методы (methods)
* class entry

---

## 8. Где хранятся методы

Методы:

* не копируются в каждом объекте
* хранятся в class definition

---

## 9. Доступ к свойствам

```php id="m9k1w2"
$user->name = "Daniil";
```

PHP:

* находит объект
* ищет свойство
* записывает значение

---

## 10. Жизненный цикл объекта

1. создание (new)
2. использование
3. удаление (refcount → 0)

---

## 11. Связь с GC

Объекты:

* могут участвовать в циклах
* очищаются через GC

---

# Ключевые вопросы + ответы

## 1. Как хранится объект в PHP?

Ответ:

zval хранит указатель на объект и object handle.

---

## 2. Что такое object handle?

Ответ:

Это уникальный идентификатор объекта.

---

## 3. Передаются ли объекты по ссылке?

Ответ:

Да, по умолчанию объекты ведут себя как ссылки.

---

## 4. Что произойдёт здесь?

```php id="q8v1c2"
$a = new stdClass();
$b = $a;

$b->x = 10;
```

Ответ:

* $a и $b указывают на один объект
* оба увидят изменение

---

## 5. Как скопировать объект?

Ответ:

Через clone

---

## 6. В чём разница между массивом и объектом?

Ответ:

Массив:

* Copy-On-Write
* копируется при изменении

Объект:

* всегда по ссылке
* не копируется

---

## 7. Где хранятся методы?

Ответ:

В class definition, а не в каждом объекте

---

## 8. Когда объект удаляется?

Ответ:

Когда refcount становится 0

---

## 9. Как объекты связаны с GC?

Ответ:

GC очищает циклические ссылки между объектами

---

## 10. Почему это важно для backend?

Ответ:

Потому что:

* объекты используются везде (Laravel, Symfony)
* влияет на память
* важно для архитектуры

---

# Мини-схема

zval → pointer → object
object → properties + class

---

# Мини-итог

Объекты в PHP хранятся как ссылки на структуру в памяти.
Они не копируются при присваивании, а передаются по ссылке.
Методы хранятся отдельно, а свойства — внутри объекта.
Понимание этого важно для работы с памятью и архитектурой.
