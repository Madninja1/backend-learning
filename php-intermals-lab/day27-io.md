# Day 27 — PHP Internals (I/O & Streams)

file: php-internals-lab/day27-io.md

## Тема

Как PHP работает с вводом/выводом и файлами (I/O)

---

## 1. Что такое I/O

I/O (Input/Output):

* чтение данных (input)
* запись данных (output)

---

## Примеры

* файлы
* сеть (HTTP)
* базы данных
* stdin / stdout

---

## 2. Работа с файлами

```php id="k2m7x1"
$file = fopen("test.txt", "r");
$data = fread($file, 1024);
fclose($file);
```

---

## 3. Что происходит внутри

PHP:

1. вызывает системный вызов (OS)
2. читает данные с диска
3. возвращает в PHP

---

## 4. Почему I/O медленный

Потому что:

* диск медленнее памяти
* есть системные вызовы
* есть задержки

---

## 5. Streams (потоки)

PHP использует streams для работы с I/O.

---

## Пример

```php id="x9v3k2"
file_get_contents("file.txt");
```

→ внутри используется stream

---

## 6. Типы streams

* file://
* http://
* php://input
* php://output

---

## 7. php://input

```php id="m7p2k4"
$data = file_get_contents("php://input");
```

→ читает тело HTTP-запроса

---

## 8. php://output

```php id="z1x8c5"
echo "Hello";
```

→ пишет в output stream

---

## 9. Буферизация (buffering)

PHP может:

* сначала накопить данные
* потом отправить

---

## Пример

```php id="t5m9k3"
ob_start();
echo "Hello";
ob_end_flush();
```

---

## 10. Почему buffering важен

* уменьшает количество I/O операций
* ускоряет вывод

---

## 11. Чтение больших файлов

```php id="y3k7x2"
while (!feof($file)) {
    fread($file, 1024);
}
```

→ читаем по частям

---

## 12. Проблемы I/O

* блокирующие операции
* задержки
* высокая стоимость

---

## 13. Как оптимизировать I/O

* читать по частям
* использовать buffering
* избегать лишних операций

---

# Ключевые вопросы + ответы

## 1. Что такое I/O?

Ответ:

Это ввод и вывод данных (файлы, сеть и т.д.)

---

## 2. Почему I/O медленный?

Ответ:

Потому что работает через диск и системные вызовы

---

## 3. Что такое stream?

Ответ:

Это абстракция для работы с вводом/выводом

---

## 4. Что делает file_get_contents?

Ответ:

Читает данные через stream

---

## 5. Что такое php://input?

Ответ:

Поток для чтения HTTP-запроса

---

## 6. Что такое php://output?

Ответ:

Поток для вывода данных

---

## 7. Что такое buffering?

Ответ:

Это накопление данных перед отправкой

---

## 8. Почему buffering полезен?

Ответ:

Снижает количество операций I/O

---

## 9. Как читать большие файлы?

Ответ:

Частями (chunked reading)

---

## 10. Почему это важно для backend?

Ответ:

Потому что:

* работа с файлами и сетью
* влияет на производительность API
* важно для обработки данных

---

# Мини-схема

PHP → stream → OS → диск / сеть

---

# Мини-итог

PHP работает с вводом/выводом через streams.
I/O — медленная операция, поэтому важно использовать buffering и читать данные эффективно.
Понимание I/O помогает писать быстрые и стабильные backend-приложения.
