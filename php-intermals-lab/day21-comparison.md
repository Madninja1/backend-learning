# Day 21 — PHP Internals (Comparison: == vs ===)

file: php-internals-lab/day21-comparison.md

## Тема

Как PHP сравнивает значения и чем отличаются == и ===

---

## 1. Два типа сравнения

### == (loose comparison)

* сравнение с приведением типов

---

### === (strict comparison)

* сравнение без приведения типов
* проверяется и значение, и тип

---

## 2. Пример

```php id="m8x2q1"
5 == "5"   // true
5 === "5"  // false
```

---

## 3. Как работает ==

PHP:

1. пытается привести типы
2. сравнивает значения

---

## Пример

```php id="y3w7k2"
"10" == 10
```

→ строка приводится к числу

→ результат: true

---

## 4. Как работает ===

PHP:

* сравнивает без преобразований

---

## Пример

```php id="v9p4c6"
"10" === 10
```

→ типы разные → false

---

## 5. Сравнение разных типов

```php id="r2k8h3"
0 == false      // true
"" == false     // true
"0" == false    // true
```

---

## 6. Опасные кейсы

```php id="t6z3m9"
"abc" == 0  // true
```

Почему:

* строка → число
* "abc" → 0

---

## 7. Сравнение null

```php id="w1n5v8"
null == false   // true
null === false  // false
```

---

## 8. Сравнение массивов

```php id="x8q4y2"
[1,2] == [1,2]   // true
[1,2] === [1,2]  // true
```

Но:

```php id="p3m7d1"
[1,2] == ["1","2"]   // true
[1,2] === ["1","2"]  // false
```

---

## 9. Сравнение объектов

```php id="k4v9z7"
$a = new stdClass();
$b = new stdClass();
```

```php id="c7n2q5"
$a == $b   // true (если свойства одинаковые)
$a === $b  // false (разные объекты)
```

---

## 10. Почему это важно

Loose comparison:

* удобный
* но опасный

Strict comparison:

* безопасный
* предсказуемый

---

# Ключевые вопросы + ответы

## 1. В чём разница между == и ===?

Ответ:

== сравнивает значения с приведением типов
=== сравнивает и тип, и значение без преобразований

---

## 2. Почему 5 == "5"?

Ответ:

PHP приводит строку к числу

---

## 3. Почему 0 == false?

Ответ:

Оба приводятся к boolean false

---

## 4. Что произойдёт здесь?

```php id="g5p8k2"
"abc" == 0
```

Ответ:

true, потому что строка приводится к числу 0

---

## 5. Что безопаснее использовать?

Ответ:

===, потому что нет неявных преобразований

---

## 6. Как сравниваются массивы?

Ответ:

== сравнивает значения
=== сравнивает и типы элементов

---

## 7. Как сравниваются объекты?

Ответ:

== сравнивает свойства
=== сравнивает ссылку (тот же объект или нет)

---

## 8. Что произойдёт здесь?

```php id="f2n6v9"
null == false
```

Ответ:

true

---

## 9. Почему == опасен?

Ответ:

Потому что приводит типы и может дать неожиданный результат

---

## 10. Почему это важно для backend?

Ответ:

Потому что:

* влияет на логику
* может вызывать баги
* важно для безопасности

---

# Мини-схема

== → приведение типов → сравнение
=== → без преобразований → сравнение

---

# Мини-итог

PHP поддерживает два типа сравнения: == и ===.
== приводит типы и может давать неожиданные результаты,
=== сравнивает строго и безопаснее.
В backend почти всегда нужно использовать ===.
